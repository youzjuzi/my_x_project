# X.1 系统架构设计

## X.1.1 总体架构概述

### 系统整体数据流向

本系统采用前后端分离、AI 服务独立部署的三层微服务架构，整体数据流向如下：

用户通过浏览器或桌面客户端启动摄像头，原始视频帧经由 WebRTC 点对点信道或 WebSocket 二进制帧传输协议，实时投递至 AI 推理服务。AI 推理服务对每一帧执行两阶段级联检测：首先由手部定位模型（YOLOv5s）在全帧范围内输出手部边界框，继而将各手部感兴趣区域（ROI）裁剪后送入手语识别模型（YOLOv5m）完成字符分类；对于视觉特征高度相似的字母组（P/Q、M/N/T、I/J、D/Z），系统自动切换至 MediaPipe HandLandmarker 提取 21 个三维手部关键点，并通过几何算法或轨迹分析作出精细判决。识别结果以 JSON 格式携带标注帧经原路返回前端渲染。

累积的离散字符序列进入拼音转汉字流水线，经 DAG 动态规划算法生成候选词，最终由 DeepSeek 大语言模型对句子进行语义润色，形成语法完整的中文表达。完成识别的文本内容通过 HTTP 接口提交至 Java 后端，持久化到 MySQL 数据库，并通过 Redis 向前端异步推送润色结果。

整体数据流可概括为：

> **摄像头视频帧 → 实时传输（WebRTC / WebSocket）→ 两阶段 YOLO 推理 → MediaPipe 精细分类（按需）→ 拼音转汉字 → LLM 语义润色 → 前端展示 / Java 后端持久化**

### 系统架构图绘制建议

建议采用**自上而下四层架构图**，各层内容如下：

---

**第一层：表现层（Presentation Layer）**

> 客户端设备、用户界面

- Web 浏览器 / Tauri 桌面客户端
- Vue 3 前端应用（手语识别页、调试页、学习模块、系统管理）
- 摄像头采集模块（调用浏览器 `getUserMedia` API）

---

**第二层：业务服务层（Business Service Layer）**

> 分左右两个子系统并列呈现，用双向箭头表示交互

- **AI 推理服务**（Python / FastAPI，端口 8001/8002）
  - WebSocket 帧接收模块
  - WebRTC 信令与媒体接收模块
  - 两阶段 YOLO 检测器（手部定位 + 字符识别）
  - MediaPipe 混合检测器（精细关键点分类）
  - 动态手势会话管理器（I/J、D/Z 轨迹分析）
  - 双手命令识别器
  - 拼音转汉字引擎
  - LLM 语义润色接口（调用 DeepSeek API）

- **Java 业务后端**（Spring Boot，端口 8080）
  - 用户认证与鉴权模块（JWT + Spring Security）
  - RBAC 权限管理
  - 识别记录持久化接口（`/sign/submit`）
  - 题库与挑战系统

---

**第三层：缓存与消息层（Cache & Messaging Layer）**

> 横跨 AI 服务与 Java 后端之间

- **Redis**：JWT Token 缓存、识别结果临时暂存、润色结果异步推送通道

---

**第四层：数据持久层（Data Persistence Layer）**

- **MySQL 数据库**：用户表、角色表、菜单表、题目表、挑战记录表、手语识别记录表（`detections`）
- **模型权重文件系统**：YOLOv5 `.pt` 权重文件（手部检测、数字识别、字母识别）、MediaPipe `.task` 模型文件

---

**连接关系说明（箭头标注建议）：**

| 连接 | 协议/方式 |
|------|-----------|
| 前端 ↔ AI 推理服务 | WebRTC（视频帧）/ WebSocket（JSON 结果） |
| 前端 ↔ Java 后端 | HTTP/HTTPS（RESTful API，携带 JWT） |
| AI 推理服务 ↔ Java 后端 | HTTP（识别文本提交 / 润色结果回调） |
| AI 推理服务 ↔ Redis | TCP（结果暂存与推送） |
| AI 推理服务 ↔ DeepSeek | HTTPS（LLM API 调用） |
| Java 后端 ↔ Redis | TCP（Token 缓存） |
| Java 后端 ↔ MySQL | JDBC（MyBatis Plus ORM） |
| AI 推理服务 ↔ 模型文件 | 本地文件系统读取（启动时懒加载） |

## X.1.2 表现层设计

本系统表现层采用 Vue 3、Vite 与 Element Plus 技术栈进行开发，构建前后端分离的 Web 用户界面。其中，Vue 3 负责页面组件化开发与响应式更新，Vite 用于提升项目构建与开发效率，Element Plus 提供统一的界面组件与交互风格。系统同时结合 Vue Router 实现前端路由管理，使用 Pinia 管理用户信息、权限状态及页面缓存状态，并通过 Axios 与 Spring Boot 业务后端及 Python/FastAPI AI 服务完成数据交互。该技术方案既能够满足后台管理类页面的开发需求，也能够支撑实时识别场景下较强的交互能力。

从界面组成上看，系统表现层主要包括用户认证界面、系统管理界面、学习训练界面、实时手语识别界面以及历史记录与个人信息界面。其中，用户认证界面负责完成登录与身份校验，系统管理界面承担用户、角色、菜单及日志等后台管理功能，学习训练界面用于支撑题库、题集、挑战和词汇练习等业务内容，历史记录与个人信息界面用于展示用户识别历史、学习活动及个人资料。整体页面组织采用典型的后台管理式布局结构，由顶部导航栏、左侧菜单栏和主内容区共同构成，从而实现不同功能模块的统一组织与切换。

实时手语识别界面是表现层设计的重点。该界面主要由顶部控制区、左侧视频识别区和右侧交互结果区构成。顶部控制区用于展示识别模式、摄像头状态和连接状态，并提供模式切换、启动与停止识别等操作；左侧视频识别区用于展示浏览器采集的实时视频画面及 AI 服务返回的检测框、识别状态、帧率和延迟信息；右侧交互结果区用于展示识别字符流、拼音缓冲、候选词及最终句子，并提供清空、复制和提交润色等交互功能。总体而言，本系统表现层具有结构清晰、模块划分明确和实时交互能力较强等特点，能够较好地满足系统管理与手语识别两类业务场景的界面需求。
